# CONSTITUTION.md — AI Finance Management

> 모든 문서 중 최상위 권위. 에이전트와 사람 모두 이 원칙에 따라 행동한다.
> **개정 실행은 Owner만** 가능하다.

---

## 1. 프로젝트 목적

가족의 가계부·자산·배당 데이터를 기반으로
AI가 자연어로 재정 분석·조회를 제공하는 **가족 전용 웹 서비스**.

구글시트로 관리하던 가계부를 Supabase로 이전하고,
대시보드 + AI 질의응답을 한 화면에서 제공한다.

---

## 2. 문서 위계

충돌 시 상위 문서가 우선한다.

```

  ↓

  ↓

  ↓

product-specs/*.md       ← 기능 상세 스펙
  ↓
ARCHITECTURE.md          ← 기술 구현 참조
```

---

## 3. 에이전트 자율 범위

`AGENTS.md`의 **"자율 결정 가능"** 항목 내에서만 사람 승인 없이 결정한다.
명시되지 않은 것은 자율 범위가 아니다.

---

## 4. 충돌 해결

- **에이전트 간 충돌**: Reviewer는 품질 기준 미달 시 거부권 행사 가능. 그 외는 Owner가 최종 결정.
- **에이전트 ↔ 문서 충돌**: 임의 결정 금지. Owner에게 보고 후 지시.
- **문서 간 충돌**: 위계 순서에 따름. 해결 안 되면 Owner에게 보고.

---

## 5. 불변 원칙

어떤 상황에서도, 어떤 에이전트도 어길 수 없다.

### 원칙 1 — 금융 데이터 무결성

실제 거래 데이터(transactions, assets, dividend)는 **절대 임의로 수정·삭제하지 않는다.**
삭제는 soft-delete(`deleted_at` 설정)만 허용한다. Hard delete 쿼리 직접 실행은 금지.

검증 방법: 코드 전체에서 `.delete()` 직접 호출 없음. 삭제 API는 `update({ deleted_at })` 방식.

### 원칙 2 — 가족 전용 접근

Owner·Spouse 두 사람만 접근한다.
**외부 공개, 다중 테넌트, 제3자 소셜 로그인 추가 금지.**

검증 방법: Supabase Auth에 허용 이메일 whitelist 유지. 신규 가입 차단.

### 원칙 3 — AI 에이전트는 생활 비서이지 투자 자문사가 아니다

**해야 하는 것:**
- "지난달 외식비 얼마야?", "올해 경조사 총 얼마?" 같은 데이터 조회에 정확히 답한다.
- "저축률이 목표에 맞게 가고 있어?" 같은 재정 현황 파악을 돕는다.

**하지 않아야 하는 것:**
- 특정 종목 매수/매도 직접 추천하지 않는다.
- 세무 신고 방법을 단언하지 않는다.
- DB에 없는 정보를 추측으로 답하지 않는다.

**면책 문구 적용 기준:**
- 일반 데이터 조회·분석 → 면책 문구 불필요
- 투자 판단·세무 등 전문 영역 질문 포함 시에만 → 답변 말미에 1줄:
  `"투자·세무 관련 최종 결정은 전문가와 확인하세요."`

### 원칙 4 — Supabase가 Source of Truth

신규 데이터의 입력 창구는 **Admin 웹 단독**이다.
구글시트는 보조 아카이브 역할이며, Supabase 데이터를 직접 편집하지 않는다.

**운영 원칙:**
- 구글시트에서 직접 편집하지 않는다.
- 월말 자동 백업은 현재 홀딩 상태 (2026-04-28 결정). 필요 시 구현.
- 불일치 발견 시 Supabase에서만 수정한다.

검증 방법: Admin 웹 이외의 경로로 데이터가 입력되는 코드 없음.

---

## 6. 개정 절차

- 개정 실행은 Owner만 가능
- 에이전트는 개정 **제안**만 가능 (이유 명시 필수)
- 헌법과 현실의 모순 발견 시 에이전트는 **반드시 보고**
